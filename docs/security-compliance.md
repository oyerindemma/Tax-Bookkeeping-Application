# Security and Compliance

## Regulatory Baseline (Nigeria)
- Data protection must align with Nigeria's data protection regime, including the Nigeria Data Protection Act (NDPA) and the Nigeria Data Protection Commission (NDPC) oversight framework.
- Tax compliance workflows must account for federal and state tax obligations and reporting expectations.
- Any payment handling must follow applicable financial regulation and payment security requirements.

## Security Standards and Frameworks
We will align controls and evidence with the following frameworks:
- ISO/IEC 27001 aligned Information Security Management System
- NIST Cybersecurity Framework for risk management and control mapping
- OWASP ASVS for web application security
- OWASP MASVS for mobile application security
- PCI DSS if payment card data is processed or stored
- SOC 2 readiness for enterprise assurance

## Security Controls (High Level)
- Encryption in transit and at rest for all sensitive data
- Role-based access control and least privilege
- Secure audit logging with immutable trails
- Data minimisation and retention rules
- Secure backup and disaster recovery
- Regular vulnerability scanning and penetration testing
- Mobile device security for offline data

## Privacy Principles
- Purpose limitation and explicit consent
- Clear data ownership and portability
- Transparent retention and deletion policies
- Privacy by design and default

## Compliance Deliverables
- Data protection impact assessment templates
- Security policies and standard operating procedures
- Incident response plan and breach notification workflow
- Vendor and third-party risk assessments
